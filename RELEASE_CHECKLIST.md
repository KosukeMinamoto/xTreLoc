# リリース前チェックリスト

## 必須項目

### 1. バージョン管理
- [ ] `build.gradle` と `pom.xml` のバージョンを `1.0-SNAPSHOT` から `1.0.0` に変更
- [ ] すべてのJARファイルにバージョン情報が正しく埋め込まれていることを確認
- [ ] リリースタグを作成（例: `v1.0.0`）

### 2. ドキュメンテーション
- [ ] `CHANGELOG.md` を作成・更新
- [ ] `README.md` のバージョン情報を更新
- [ ] ユーザーマニュアルの確認

### 3. ビルドとテスト
- [ ] クリーンビルドの実行（`./gradlew clean build` または `mvn clean package`）
- [ ] GUI版とCLI版の両方が正常にビルドされることを確認
- [ ] 基本的な機能テストの実行
- [ ] デモデータセットでの動作確認

### 4. リリースアセットの準備
- [ ] GUI版JARファイル（`xTreLoc-GUI-1.0.0.jar`）
- [ ] CLI版JARファイル（`xTreLoc-CLI-1.0.0.jar`）
- [ ] macOS用 `.app` バンドル（オプション）
- [ ] macOS用 `.dmg` インストーラー（オプション）
- [ ] Javadoc JAR（オプション）

### 5. GitHubリリースの作成
- [ ] リリースページでタグを作成（例: `v1.0.0`）
- [ ] リリースタイトルを設定
- [ ] リリースノートを記入（`CHANGELOG.md`から生成可能）
- [ ] アセットファイルをアップロード
- [ ] プレリリースではなく正式リリースとして公開

### 6. コード品質
- [ ] コンパイル警告の確認と修正
- [ ] 未使用コードの削除
- [ ] コメントの整備

## 推奨項目

### セキュリティ
- [ ] 依存関係の脆弱性スキャン
- [ ] 機密情報のハードコーディング確認

### パフォーマンス
- [ ] メモリリークの確認
- [ ] 起動時間の確認

### ユーザビリティ
- [ ] エラーメッセージの確認
- [ ] ヘルプドキュメントの確認
